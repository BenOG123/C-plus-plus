C++ Commands for Unreal Engine

Essential C++ commands and functions for game development in Unreal Engine

Actor and Component Commands

CreateDefaultSubobject<T>()
Creates a new component and attaches it to the actor.
UStaticMeshComponent* MeshComponent = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("MeshComponent"));

How to use:

  1. Use in the constructor of your actor class.
  2. Specify the type of component you want to create as the template parameter.
  3. Provide a unique name for the component as a parameter.
  4. Assign the result to a pointer of the appropriate type.
  5. After creation, you can set properties of the component.
GetWorld()
Returns a pointer to the current game world.
UWorld* World = GetWorld(); if (World) { // Perform world-related operations }

How to use:

  1. Can be called from any UObject-derived class.
  2. Use to access the game world for various operations.
  3. Always check if the returned pointer is valid before using it.
  4. Useful for spawning actors, setting timers, and more.
SpawnActor<T>()
Spawns a new actor of the specified class in the world.
AMyActor* SpawnedActor = GetWorld()->SpawnActor<AMyActor>(AMyActor::StaticClass(), SpawnLocation, SpawnRotation);

How to use:

  1. Call on the UWorld object, usually obtained via GetWorld().
  2. Specify the class of the actor you want to spawn as the template parameter.
  3. Provide the class, spawn location, and rotation as parameters.
  4. Can also use FActorSpawnParameters for more advanced spawn options.
  5. Check if the returned pointer is valid before using the spawned actor.

Movement and Transform Commands

SetActorLocation()
Sets the actor's position in the world.
SetActorLocation(FVector(100.0f, 200.0f, 300.0f));

How to use:

  1. Call on any AActor-derived class.
  2. Provide an FVector with the desired x, y, and z coordinates.
  3. Can be used in Tick() for continuous movement.
  4. Consider using SetActorLocationAndRotation() if you need to set both at once.
AddActorLocalRotation()
Adds a delta rotation to the actor in its local space.
AddActorLocalRotation(FRotator(0.0f, 5.0f, 0.0f));

How to use:

  1. Call on any AActor-derived class.
  2. Provide an FRotator with the desired pitch, yaw, and roll.
  3. Useful for rotating an actor relative to its current orientation.
  4. Can be used in Tick() for continuous rotation.
GetActorForwardVector()
Returns the forward (X) unit vector of the actor in world space.
FVector ForwardVector = GetActorForwardVector(); SetActorLocation(GetActorLocation() + ForwardVector * 100.0f);

How to use:

  1. Call on any AActor-derived class.
  2. Returns an FVector representing the forward direction.
  3. Useful for moving actors in their facing direction.
  4. Can be combined with other vectors for complex movements.

Input Commands

BindAxis()
Binds a function to an axis input.
InputComponent->BindAxis("MoveForward", this, &AMyCharacter::MoveForward);

How to use:

  1. Use in the SetupPlayerInputComponent() function of your character class.
  2. First parameter is the name of the axis as defined in the project settings.
  3. Second parameter is the object to call the function on (usually 'this').
  4. Third parameter is a pointer to the function to be called.
  5. The bound function should take a single float parameter.
BindAction()
Binds a function to an action input.
InputComponent->BindAction("Jump", IE_Pressed, this, &ACharacter::Jump);

How to use:

  1. Use in the SetupPlayerInputComponent() function of your character class.
  2. First parameter is the name of the action as defined in the project settings.
  3. Second parameter is the key event (IE_Pressed, IE_Released, IE_Repeat).
  4. Third parameter is the object to call the function on (usually 'this').
  5. Fourth parameter is a pointer to the function to be called.
EnableInput()
Enables input for a specific player controller on this actor.
APlayerController* PC = Cast<APlayerController>(GetController()); if (PC) { EnableInput(PC); }

How to use:

  1. Call on the actor that should receive input.
  2. Pass a pointer to the player controller as the parameter.
  3. Typically used when you want non-player actors to receive input.
  4. Make sure to call DisableInput() when input is no longer needed.

Collision and Physics Commands

SetCollisionProfileName()
Sets the collision profile for a component.
GetCapsuleComponent()->SetCollisionProfileName(TEXT("Pawn"));

How to use:

  1. Call on any UPrimitiveComponent-derived class.
  2. Provide the name of a collision profile defined in the project settings.
  3. Useful for setting up collision behavior for actors or components.
  4. Common profiles include "Pawn", "BlockAllDynamic", "OverlapAllDynamic", etc.
AddImpulse()
Adds an impulse to a physics-enabled component.
GetMesh()->AddImpulse(FVector(0.0f, 0.0f, 1000.0f));

How to use:

  1. Call on any UPrimitiveComponent with physics simulation enabled.
  2. Provide an FVector representing the impulse direction and magnitude.
  3. Useful for instant forces like explosions or jumps.
